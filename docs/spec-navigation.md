# ナビゲーションバー 外部仕様

## 概要

左側固定のサイドバーにナビゲーションメニューを配置する。  
ナビゲーションはトップレベルのセクションと、セクション内のサブページで構成される。

---

## 構造

```
AzDoDeck
Azure DevOps
─────────────────────────────
▼ Pull Requests              ← セクション見出し（クリック不可）
    Search                   ← サブページ
    My Reviews               ← サブページ
  Work Items                 ← トップレベルページ
  Commits                    ← トップレベルページ
─────────────────────────────
  Settings                   ← トップレベルページ（常時アクセス可能）
```

---

## セクション（Section）

現在 Pull Requests のみセクション構造を持つ。

| 項目 | 仕様 |
|------|------|
| 表示 | アイコン ＋ ラベルの見出し行として表示。それ自体はクリックできない |
| サブページ | セクション見出しの下に左インデント ＋ 縦線で表示 |
| 無効状態 | Organization 未設定時は全体が薄く表示される |

### Pull Requests セクション内サブページ

| サブページ | 説明 |
|------------|------|
| **Search** | PR のキーワード検索。条件（ステータス・Organization）を指定して手動検索 |
| **My Reviews** | 自分がレビュワーとして割り当てられている PR を一覧表示 |

---

## トップレベルページ

セクションに属さない単独のナビゲーション項目。

| ページ | アイコン | 説明 |
|--------|----------|------|
| **Work Items** | ListChecks | Work Item の検索 |
| **Commits** | GitCommit | コミットの検索 |
| **Settings** | Settings | Organization の追加・管理 |

---

## アクティブ状態

- 現在表示中のページのアイテムは背景色（secondary）で強調表示される
- セクション見出し自体はアクティブ状態を持たない

---

## 無効状態

- Settings を除くすべてのページ・サブページは、Organization が 1 件も設定されていない場合は無効（disabled）になる
- Organization 未設定時は自動的に Settings ページが表示される

---

## 初期ページ

アプリ起動時に Organization が設定済みの場合、**Pull Requests > Search** ページを表示する。

---

## 将来の拡張候補

| 候補 | 概要 |
|------|------|
| Work Items にサブページ追加 | 「My Work Items」「Assigned to Me」など |
| Commits にサブページ追加 | 「My Commits」など |
| My Reviews にバッジ表示 | 未投票 PR 件数をナビゲーション上にバッジで表示 |
| セクションの折りたたみ | セクション見出しクリックでサブページを展開／折りたたみ |
