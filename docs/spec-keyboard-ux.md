# 仕様書: キーボード / UX (カテゴリ E)

作成日: 2026-06-18 / ステータス: 提案 (未着手)

## 概要

DevDeck はキーボード中心の運用を hard requirement としている (AGENTS.md)。既に
コマンドパレット (`Ctrl+K`)、Linear 風の `G` チェーン遷移、フォーカス移動
(`Ctrl+G/P/N`)、グリッド単文字ショートカットを備える。

このカテゴリは、キーボード操作の到達範囲と一貫性をさらに広げ、マウスに頼らず
すべての日常操作を完結できるようにする機能群をまとめる。

## 対象項目

| # | 項目 | 概要 |
|---|---|---|
| E-33 | ショートカットのカスタマイズ UI | キーバインドを設定画面で変更可能に |
| E-34 | アイテム履歴 / 戻る・進む | 直近開いたアイテムをブラウザ的に戻る・進む |
| E-35 | グリッドの複数行選択と一括アクション | 範囲/トグル選択 → 状態・担当などを一括変更 |
| E-36 | コマンドパレットからの直接アクション拡張 | PR 承認・WI 状態変更等をパレットから実行 |

---

## E-33 ショートカットのカスタマイズ UI

### 目的
ユーザーが既存の OS / 他ツールと衝突するキーを再割り当てできるようにする。

### 方針
- 現状ショートカットは `App.tsx` の `commandActions` とグローバル keydown に
  ハードコードされている。これをキー定義のレジストリ (id → 既定キー) に集約し、
  ユーザー上書きを `localStorage` (`azdodeck:keybindings:v1`) に保存する。
- 設定画面に一覧 + 再割り当て UI を追加。キャプチャ中のキー入力を拾い、衝突を検出して
  警告する。リセットで既定に戻す。
- `HelpDialog` の表示はレジストリから動的生成し、カスタマイズが反映されるようにする。
- グローバルホットキー (ウィンドウ表示) は Tauri 側設定 (`update_app_settings`) で既に
  扱っているため、それと整合させる (アプリ内キー = localStorage、OS グローバル = 設定)。

### 受け入れ基準
- 設定画面でショートカットを変更でき、即座に反映される。
- 衝突するキーに警告が出る。
- HelpDialog がカスタマイズ後のキーを表示する。
- リセットで既定に戻る。

## E-34 アイテム履歴 / 戻る・進む

### 目的
パレットの recent (`azdodeck:commandPalette:recentItems`) とは別に、開いたアイテムを
ブラウザ的に「戻る・進む」できるようにし、調査中の行き来を高速化する。

### 方針
- ビュー + 選択アイテム (PR / 作業項目 / コミット) を 1 つのナビゲーション履歴スタックで
  管理する。`Alt+Left` / `Alt+Right` で戻る・進む。
- 履歴エントリは「ビュー種別 + 組織 + 対象 id + 検索 query」。再現に必要な最小情報のみ
  保持し、データ自体は TanStack Query から再取得する。
- セッション内メモリで十分 (永続化は不要)。スタック上限を設ける。

### 受け入れ基準
- アイテムを複数開いた後、`Alt+Left` で直前の選択に戻れる。
- `Alt+Right` で進める。
- 戻り先のビュー・選択・検索状態が復元される。

## E-35 グリッドの複数行選択と一括アクション

### 目的
作業項目グリッドで複数行を選び、状態・担当・優先度をまとめて変更する。バックエンドは
既に一括対応 (`set_work_items_state` / `assign_work_items` / `set_work_items_priority` が
複数 id を受ける) なので、UI 側の選択モデルを整備する。

### 方針
- グリッドに選択集合を追加。`Shift+↑/↓` で範囲選択、`Space` でトグル、`Ctrl+A` で全選択。
- 選択中は一括アクションバー (件数表示) を出し、既存の単文字ショートカット
  (`S` 状態 / `A` 担当 / `P` 優先度) を選択全行に適用。
- 選択と単一行ナビゲーションのフォーカス挙動を両立させる (行移動を奪わない)。
- 一括変更も既存のステージング → `Ctrl+S` 適用フローに乗せる。

### 受け入れ基準
- キーボードのみで複数行を選択できる。
- 状態/担当/優先度の一括変更が選択全行に適用される。
- 選択解除・全選択がキーボードで行える。
- 単一行ナビゲーションが従来どおり機能する。

## E-36 コマンドパレットからの直接アクション拡張

### 目的
現在パレットの Work Items アクションは選択中アイテムへの操作 (状態変更等) に限られ、
PR 側のアクションはない。パレットを「移動」だけでなく「実行」のハブにする。

### 方針
- 選択中の PR に対するアクションをパレットに追加: 承認 / 却下 / 待機、スレッド解決、
  ブラウザで開く。`PrReviewService` の既存コマンド (`submit_pull_request_vote`,
  `set_pull_request_thread_status`) を再利用。
- アクションの `disabled` は現在のビューと選択状態で制御 (既存 Work Items アクションと
  同パターン)。
- 破壊的になり得る操作 (却下等) は確認を挟む。

### 受け入れ基準
- PR 選択中にパレットから承認/却下/待機が実行できる。
- 選択がない / 対象外ビューではアクションが無効化される。
- 実行後に該当クエリが invalidate され UI に反映される。

## E-37 サイドバー最上位ナビの並べ替え

### 目的
利用頻度に合わせて左ナビの最上位項目 (Pull Requests / Work Items / Commits /
Pipelines / Code) の順序をユーザーが自由に変えられるようにする。

### 方針
- 順序は `NavEntryId[]` の state で保持し、localStorage
  (`azdodeck:layout:navOrder`) に永続化。読み込み時は既知 id のみ採用し、欠けた id は
  既定順で末尾補完 (`normalizeNavOrder`) して将来の項目追加に耐える。
- 並べ替えロジックは純粋関数 `reorderNav` に集約し、D&D とキーボードの両経路が再利用。
- マウス: HTML5 Drag & Drop (`dragstart` / `dragover` で `preventDefault` / `drop`)。
- キーボード: フォーカス中の項目を `Alt+↑` / `Alt+↓` で移動 (素の矢印=フォーカス移動より
  優先)。安定した `key` により並べ替え後もフォーカスは同じ項目に残る。
- Help / Settings は下部固定で対象外。

### 受け入れ基準
- D&D で最上位項目を並べ替えられ、リロード後も順序が復元される。
- `Alt+↑` / `Alt+↓` でマウスなしに並べ替えられ、フォーカスが移動した項目に残る。
- 保存データが壊れていても既定順にフォールバックする。

---

## 共通の非機能要件
- すべての新規 UI はキーボードのみで開く・操作する・閉じる経路を持つこと
  (AGENTS.md: open focused / contain keys / restore focus on close)。
- ショートカットは HelpDialog で discoverable にすること。
- 行ナビゲーション中にフォーカスを奪わないこと。
- 検証: `pnpm tsc --noEmit`, `pnpm test -- --run`, リスクに応じ `pnpm test:e2e`。
