# 仕様書: My Reviews（レビュー待ち一覧）

## 概要

自分がレビュワーとしてアサインされているアクティブな Pull Request を一覧表示する専用ビュー。
既存の "Pull Requests"（汎用検索）とは独立して、サイドバーに **"My Reviews"** として追加する。

ビューを開いた瞬間に自動ロードし、常に「今レビューが必要なもの」をすぐ把握できることを目的とする。

スコープ: 選択中の Organization に属する **全 Project を横断**して表示する。

---

## 必須要件

| # | 要件 |
|---|------|
| R1 | 自分がレビュワーになっているアクティブな PR をグリッドで表示する |
| R2 | カラム: PR 番号、リポジトリ、タイトル、作成者、作成日、ターゲットブランチ、必須/オプション、自分の投票状態 |
| R3 | `Enter` キーで選択中の PR をデフォルトブラウザで開く |

---

## 確定済み要件

### UX / インタラクション

| # | 要件 | 理由 |
|---|------|------|
| P1 | ビューを開いた瞬間に自動ロード（Search ボタン不要） | "インボックス" として機能させるため |
| P2 | `↑` / `↓` キーで行選択を移動 | キーボードだけで操作を完結させるため |
| P3 | `R` キーまたはリフレッシュボタンで再取得 | 手動で最新化できるようにするため |
| P4 | サイドバーに未レビュー（No Vote）件数バッジを常時表示。**バックグラウンドで定期更新**（アプリ起動時 + 5 分間隔） | 他のビューを見ていても件数を把握できるため |
| P5 | 投票状態でのフィルタタブ。**既定は No Vote** | 再フェッチせず即座に絞り込めるため。最もアクション可能な状態をデフォルトにする |
| P6 | テキスト入力によるフィルタ（リポジトリ名・タイトル・作成者に対してインクリメンタル検索） | キーワードで素早く対象を絞れるため |
| P7 | "ドラフトを含む" チェックボックスでドラフト PR の表示/非表示を切り替え（既定: 非表示） | ドラフトを状況に応じて確認できるようにするため |

### 表示情報

| # | 要件 | 理由 |
|---|------|------|
| P8 | PR 作成者（Author）カラム | 誰から依頼されたかが一目でわかる |
| P9 | 作成日を相対表示（例: "3d"）、ホバーで絶対日時を表示 | 滞留している古い PR に気づきやすい |
| P10 | ドラフト PR に "Draft" バッジを表示 | ドラフト表示時に一目でわかる |
| P11 | ターゲットブランチをカラムとして表示 | どのブランチへのマージかを確認する |
| P12 | PR 番号カラム（`#42` 形式）。クリックで Azure DevOps を開く | 口頭で PR を参照する際の識別子。Enter キーの代替としても機能 |
| P13 | 3 日以上経過した PR の行に薄い警告背景色 | 古い PR への気づきを促進 |
| P14 | グリッド下部にステータスバー（例: "12 件中 5 件が未投票"） | フィルタ前の全体像を常に把握できる |

---

## レイアウト方針

**コンパクト表示を優先する。** ダッシュボードとして一度に多くの PR を俯瞰できることが重要。

| 項目 | 値 |
|------|-----|
| 行の高さ | コンパクト（py-1.5 程度、デフォルトの py-3 から削減） |
| セル内パディング | px-2（デフォルトの px-4 から削減） |
| フォントサイズ | text-sm 基本、PR 番号・ブランチ・日付は text-xs |
| バッジサイズ | text-xs + px-1.5 py-0.5（小さめ） |
| ヘッダー | text-xs uppercase tracking-wide で小さく |
| グリッド外余白 | 上下左右 p-2（ビュー全体の余白を最小化） |

---

## フィルタバー

グリッド上部に常時表示するフィルタバー。すべてのフィルタはフロントエンドのみで処理し、再フェッチは行わない。

| コントロール | 種別 | 動作 |
|-------------|------|------|
| テキスト入力 | インクリメンタル検索 | 入力した文字列をリポジトリ名・タイトル・作成者に対して部分一致で絞り込む（P6） |
| 投票状態タブ | 排他選択 | **No Vote（既定）** / Approved / Rejected / All の順で表示（P5） |
| ドラフトを含む | チェックボックス | ON のときドラフト PR を表示、OFF のとき非表示（既定: OFF）（P7） |

複数フィルタは AND 条件で適用する。

### 投票状態タブのグルーピング

| タブ | 対象の vote 値 | 表示順 |
|------|---------------|--------|
| **No Vote** | `0`（未投票） | 1（**既定**） |
| Approved | `10`（承認）、`5`（承認 w/ Suggestions） | 2 |
| Rejected | `-10`（却下）、`-5`（Waiting for Author） | 3 |
| All | 全件 | 4（最後） |

## グリッドカラム

左から右へ、以下の順で表示する。

| # | カラム | 幅 | 内容 |
|---|--------|-----|------|
| 1 | PR# | 固定 w-16 | `#42` 形式。クリックで Azure DevOps を開く（P12）。text-xs |
| 2 | Repository | 可変 min-w-40 | `プロジェクト名 / リポジトリ名`。text-sm |
| 3 | Title | 可変 flex-1 | PR タイトル。ドラフトの場合は "Draft" バッジを付加（P10）。text-sm、truncate |
| 4 | Author | 可変 w-28 | PR 作成者名（P8）。text-sm、truncate |
| 5 | Created | 固定 w-16 | 相対日時（P9）。text-xs。3 日以上は警告色テキスト |
| 6 | Target | 可変 w-24 | ターゲットブランチ名（`refs/heads/` 除去）。text-xs、truncate（P11） |
| 7 | Role | 固定 w-20 | `Required` または `Optional` バッジ |
| 8 | My Vote | 固定 w-28 | 投票状態バッジ（色付き） |

### 投票状態バッジ

| 状態 | ラベル | 色 |
|------|--------|-----|
| 未投票 | No Vote | グレー |
| 承認（コメント付き） | Approved w/ Suggestions | 黄緑 |
| 承認 | Approved | 緑 |
| 作成者待ち | Waiting for Author | 黄 |
| 却下 | Rejected | 赤 |

### Required / Optional バッジ

| 種別 | 色 |
|------|----|
| Required（必須） | 赤系 |
| Optional（任意） | グレー |

### 行スタイル

| 条件 | スタイル |
|------|---------|
| 通常 | 背景なし |
| 選択中（キーボード） | `bg-accent` |
| ホバー | `bg-muted/50` |
| 3 日以上経過 | `bg-orange-50 dark:bg-orange-950/20`（P13） |
| 3 日以上経過 + 選択中 | `bg-orange-100 dark:bg-orange-900/30` |

## ステータスバー（P14）

グリッド下部に固定表示。

```
12 件中 5 件が未投票  |  フィルタ適用中: 8 件表示
```

- 左側: 全件数と No Vote 件数（フィルタ前の値）
- 右側: フィルタ適用時のみ表示（フィルタ後の件数）

## データ取得

### API エンドポイント

プロジェクトレベル PR 検索を使用（リポジトリ横断）:

```
GET https://dev.azure.com/{org}/{project}/_apis/git/pullrequests
  ?searchCriteria.reviewerId={authenticatedUserId}
  &searchCriteria.status=active
  &$top=200
  &api-version=7.1
```

- 全プロジェクトを iterate し、各プロジェクトで上記エンドポイントを呼ぶ
- `searchCriteria.reviewerId` でサーバ側フィルタ（自分がレビュアーの PR のみ返る）
- `authenticatedUserId` は org 登録時に `connectionData` から取得済み

### ソート・件数制限

- **ソート**: 作成日降順（`creationDate DESC`）
- **上限**: 200 件（API の `$top` で制御）

### 複数 Organization

最初の 1 件に限定（将来の拡張候補としてセレクター切り替えを検討）。

### バッジ用バックグラウンド更新（P4）

- アプリ起動時に 1 回フェッチ
- 以後 5 分間隔で定期フェッチ
- 結果から `vote === 0` の件数をカウントし、サイドバーの My Reviews バッジに表示
- My Reviews ビューを開いている場合は、バックグラウンド更新の結果でグリッドも更新

## レスポンスマッピング

### GitPullRequest → ReviewPullRequestSummary

| バックエンド出力 | 元データ |
|---|---|
| `pullRequestId` | `GitPullRequest.pullRequestId` |
| `myVote` | `reviewers[]` から `id === authenticatedUserId` の `vote` を取得 |
| `myVoteLabel` | vote 値に応じたラベル（上記投票状態バッジ参照） |
| `myIsRequired` | 同レビュアーの `isRequired` フィールド |
| `isDraft` | `GitPullRequest.isDraft` |
| `createdBy` | `GitPullRequest.createdBy.displayName` |
| `targetRefName` | `GitPullRequest.targetRefName`（`refs/heads/` 除去済み） |
| `webUrl` | `_links.web.href` または `url` |
