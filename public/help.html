AzDoDeck v0.1.0

Azure DevOps の Pull Request・Work Item・Commit を一画面で俯瞰できるデスクトップダッシュボード。

はじめに

AzDoDeck は Azure DevOps Services（dev.azure.com）専用の読み取り専用ダッシュボードです。複数の組織・プロジェクト・リポジトリをまたいで情報を俯瞰し、必要なものだけ Web ブラウザで開くワークフローを実現します。

対応サービス

Azure DevOps Services（dev.azure.com）のみ

認証方式

Personal Access Token（PAT）または Azure CLI

データ保存

ローカル SQLite キャッシュ。シークレットは Windows Credential Manager に保存

書き込み操作

なし。すべて読み取り専用

クイックスタート

PAT 認証の場合

  1. Azure DevOps で PAT を発行 スコープ: Code (Read) Work Items (Read) Project and Team (Read)
  2. Settings → Add Organization → PAT を選択 組織名（例: mycompany）と PAT を入力して「追加」
  3. 初回同期の完了を待つ サイドバー右上のインジケーターが緑になれば完了
  4. My Reviews を開く Alt+1 でレビュー待ち PR の一覧が表示されます

Azure CLI 認証の場合

  1. Azure CLI をインストール・ログイン ターミナルで az login を実行
  2. Settings → Add Organization → Azure CLI を選択 組織名のみ入力して「追加」。トークンは自動取得されます
ヒント: PAT の有効期限が切れた場合は Settings → 対象組織の歯車アイコン → PAT を更新してください。

My Reviews

自分がレビュアーとして登録されている Pull Request の一覧です。投票状態と鮮度を一目で把握できます。

グリッドの見方

  • 投票アイコン: 承認 / 却下 / 待機 / 未投票の状態を色とアイコンで表示
  • ハイライト（黄色）: 作成から 3 日以上経過した PR（レビュー遅延の目安）
  • 列リサイズ: 列の境界をドラッグで幅を変更できます。幅は自動保存されます

Pull Request Search

プロジェクト・リポジトリ・ステータスを条件に、組織横断で PR を検索します。

フィルター

  • Project: 特定のプロジェクトに絞り込み（空 = 全プロジェクト）
  • Repository: Project を選択すると選択可能になります
  • Status: Active / Completed / Abandoned / All

My Work Items

自分にアサインされた Work Item の一覧です。テキストでのインライン絞り込みができます。

  • テキストフィルター入力欄でタイトル・ID・タイプを絞り込み
  • R キーまたは「更新」ボタンで最新データを取得
  • 行クリックまたは Enter で Azure DevOps の Work Item ページを開く

Work Item Search

プロジェクト・状態・タイプでフィルタリングして Work Item を検索します。

フィルター

  • Project: 対象プロジェクトを選択（空 = 全プロジェクト）
  • State: Active / Resolved / Closed / All など
  • Type: Bug / Task / User Story / Feature / Epic など

Review Preview

PR 番号にマッチするローカル HTML ファイルをアプリ内でプレビューできます。チームで PR レビューコメントを HTML でまとめている場合に便利です。

設定方法

  1. HTML ファイルを所定フォルダに配置 ファイル名に PR 番号を含めてください（例: review-1234.html、pr_1234_notes.html）
  2. Settings → Review Preview Folder を設定 HTML ファイルが入っているフォルダのパスを指定
  3. My Reviews で PR 行をクリック PR 番号にマッチするファイルが見つかればパネルが開きます
ヒント: ファイル名に PR 番号が含まれていれば命名規則は問いません。複数ファイルがマッチした場合は最初のファイルが使われます。

組織管理

複数の Azure DevOps 組織を登録・管理できます。

組織の追加

  1. Settings を開く Alt+6 またはサイドバー下部の Settings ボタン
  2. 「組織を追加」をクリック 認証方式（PAT / Azure CLI）を選択
  3. 必要情報を入力して追加 PAT の場合は組織名と PAT、Azure CLI の場合は組織名のみ

組織の削除

Settings → 対象組織の削除ボタンをクリックしてください。ローカルキャッシュも削除されます。認証情報（PAT）は Windows Credential Manager から自動削除されます。

セキュリティ: PAT は Windows Credential Manager に暗号化して保存されます。アプリのデータベースファイルには一切保存されません。

同期について

  • 起動時に自動で差分同期を実行します
  • ヘッダーの「同期」ボタンで手動同期できます（30 秒クールダウンあり）
  • Settings で自動同期間隔を変更できます（1 / 2 / 5 / 10 / 15 / 30 分）

Tips

列幅は自動保存される

すべてのグリッドの列幅はブラウザのローカルストレージに保存されます。アプリを再起動しても維持されます。

サイドバーの幅も調整できる

サイドバーの右端をドラッグすると幅を 220〜420 px の範囲で調整できます。ダブルクリックでデフォルト幅に戻ります。

複数組織を切り替えて使う

Settings で複数の組織を登録しておくと、各ビューの「組織」セレクターで切り替えられます。

SmartScreen 警告について

インストール時に Windows SmartScreen の警告が出る場合があります。「詳細情報」→「実行」を選択してください。コード署名は今後のリリースで対応予定です。

PAT の最小スコープ

PAT に必要な最小スコープ: Code (Read) Work Items (Read) Project and Team (Read)。最小権限で発行することを推奨します。
